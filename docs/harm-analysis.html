FREQ-TR / HARM forensic comparison

HARM: current curve vs proposed harmonic fullness

Objetivo: comprobar si el problema viene del numero total de armonicos o de como se abre la curva.

Resumen: la implementacion historica usaba serie armonica real hasta H24. La version actual usa H1..H16, con cola alta mas inclinada para que el extremo no forme un bloque compacto.

Implementacion historica

El oscilador armonico se aplica a la portadora/moduladora, no directamente al audio como saturacion.

maxHarmonics = 24
baseWeight[n] = 1 / n^1.2

density = HARM^0.75
count = 1 + density * 23
blend = HARM^0.80

weight[1] = 1
weight[n > 1] = activeFraction(n, count) * blend / n^1.2
gain = 1 / sqrt(sum(weight^2))

La entrada de cada armonico es continua por amplitud, pero el modelo sigue teniendo un count: primero se completa H2, luego H3, etc.

Implementacion propuesta/aplicada

Quitar el contador perceptual, limitar el stack util a H1..H16 y usar una envolvente continua tipo fullness/slope con gates progresivos y rolloff final.

maxHarmonics = 16
amount = HARM^0.90
slope = lerp(4.6, 2.15, HARM^0.70)
gate[n] = smoothstep((HARM - start[n]) / 0.22)
tail[n] = -38 dB * HARM * harmonicIndex^2

weight[1] = 1
weight[n > 1] = gate[n] * amount * gain(tail[n]) / n^slope
gain = 1 / sqrt(sum(weight^2))

No desaparece la serie armonica: H2..H16 existen como serie armonica, pero se abren progresivamente: H2 primero, despues H3, despues H4... y la cola final cae mas rapido.

Diagnostico

No parece un fallo de serie armonica: se usan multiplos enteros reales de la portadora: H1, H2, H3...

Si parece un fallo de densidad: con la implementacion historica, a 25% ya habia H1..H9 y a 100% H24. En frequency shifting esto genera demasiadas copias desplazadas frente al perfil observado en FREAK.

Recomendacion

Probar la propuesta H16 fullness/slope + gates progresivos + tail rolloff. Mantiene mas extension que H10, pero con una cola final mucho mas inclinada y sin levantar todo el bloque a la vez.

Si queda apagado, bajar kHarmSlopeMin de 2.15 a 2.00. Si sigue aspero, subirlo a 2.30.
